# 针刺 - 云端构建部署指南

## 🎯 目标
将项目推送到 GitHub，使用 GitHub Actions 自动构建 APK。

---

## 步骤一：创建 GitHub 仓库

### 1. 在 GitHub 网页创建
1. 打开 https://github.com/new
2. 仓库名称：`zhenci-app`（或其他名称）
3. 选择 **Public**（公开）或 **Private**（私有）
4. **不要**勾选 "Initialize this repository with a README"
5. 点击 **Create repository**

### 2. 复制仓库地址
创建后会显示类似：
```
https://github.com/你的用户名/zhenci-app.git
```

---

## 步骤二：推送代码

### 方式一：使用脚本（推荐）

```bash
cd /Users/king/Desktop/针刺-app

# 运行推送脚本
./push-to-github.sh
```

按提示操作即可。

### 方式二：手动推送

```bash
cd /Users/king/Desktop/针刺-app

# 初始化仓库
git init

# 添加所有文件
git add .

# 提交
git commit -m "Initial commit"

# 关联远程仓库（替换为你的仓库地址）
git remote add origin https://github.com/你的用户名/zhenci-app.git

# 推送到 main 分支
git branch -M main
git push -u origin main
```

---

## 步骤三：查看自动构建

推送成功后，GitHub Actions 会自动开始构建：

1. 打开你的 GitHub 仓库页面
2. 点击顶部的 **Actions** 标签
3. 查看 "Build APK" 工作流运行状态
4. 等待约 5-10 分钟

---

## 步骤四：下载 APK

### 方式一：从 Actions 下载

1. 进入 Actions 页面
2. 点击最新的成功构建
3. 滚动到页面底部的 **Artifacts**
4. 下载 `app-debug` 或 `app-release`

### 方式二：从 Releases 下载

1. 点击仓库页面的 **Releases**（右侧）
2. 找到最新版本
3. 下载 APK 文件

---

## 📱 安装到手机

### 1. 下载 APK
- 方式一：在电脑上下载，发送到手机
- 方式二：直接在手机浏览器下载

### 2. 允许安装
- 设置 → 安全 → 允许安装未知来源应用
- 找到下载的 APK，点击安装

### 3. 开始使用
- 打开应用
- 添加今日任务
- 等待定时提醒

## 🐛 常见问题

### Q: 推送失败
```bash
# 检查远程仓库
git remote -v

# 重新设置
git remote set-url origin https://github.com/你的用户名/zhenci-app.git
```

### Q: 构建失败
1. 检查 Actions 日志中的错误信息
2. 确保所有代码文件已提交
3. 检查 Gradle 配置是否正确

### Q: 无法下载 APK
- 确保使用 GitHub 账号登录
- 检查网络连接
- 尝试刷新页面
